## 🎯 このコードは何か？

### ざっくり言うと

**M5Stackという小型コンピュータに、各種センサーを接続して、部屋の環境を計測・表示するプログラムです。**

```
┌─────────────┐
│  M5Stack    │ ← 小型コンピュータ
│  (ESP32)    │
└──────┬──────┘
       │ センサーを接続
       │
       ├─ 温湿度センサー（BME280）
       ├─ 照度センサー（NJL7520L）
       └─ 内蔵IMU（加速度・地磁気）
       
       ↓
       
   ┌─────────────┐
   │  ディスプレイ  │ ← 2インチの画面に表示
   └─────────────┘
       
       ↓
       
   ┌──────────────────────┐
   │ OpenWeatherMap API   │ ← 天気情報を取得
   └──────────────────────┘
```

### 何のためのプログラムか？

**デスク上に置いて、常に環境をチェック**

- 「今日は湿度が高い。除湿しよう」
- 「気温28°C、外は晴れ。服装を決めよう」
- 「3分間使ってないな。休憩しよう」（顔アニメで通知）

こういった**ちょっとした判断**に役立つ情報を提供します。

## ⚙️ 実装の工夫

### 1. BME280マルチアドレス自動検出

**問題**：ハードウェア個体差でアドレスが異なる

**解決**：複数アドレスを自動スキャン

```cpp
bool initBME280(){
    byte addresses[] = {0x76, 0x77, 0x75};
    for(byte i=0; i<3; i++){
        if(bme.begin(addresses[i], &Wire)){
            return true;
        }
    }
    return false;
}
```

### 2. 差分描画による最適化

**問題**：小型ディスプレイへの連続描画は重い

**解決**：3段階の更新戦略

```cpp
// 画面切り替え時のみ全画面再描画
if(force || cityChanged) {
    drawWeatherFull();
}

// 説明文変更時のみ部分更新
if(description != lastDescription) {
    M5.Lcd.fillRect(20, 80, 280, 40, BG);
    M5.Lcd.printf("%s", description.c_str());
}

// 温度0.1°C以上の変化のみ更新
if(fabs(temp - lastTemp) >= 0.1f) {
    M5.Lcd.fillRect(70, 140, 200, 80, BG);
    M5.Lcd.printf("%.1f C", temp);
}
```

**効果**：描画負荷 **60-80%削減**

### 3. リングバッファによるデータ管理

**問題**：メモリ効率とアクセス速度の両立

**解決**：固定サイズバッファで循環管理

```cpp
constexpr int MAX_DATA_POINTS = 60;
float tempData[MAX_DATA_POINTS];
int dataIndex = 0;

// 新データ追加
tempData[dataIndex] = temp;
dataIndex = (dataIndex + 1) % MAX_DATA_POINTS;

// 最新データ取得
float latest = tempData[(dataIndex + 59) % 60];
```

**メリット**：
- RAM効率化
- O(1)アクセス
- ヒープフラグメンテーション回避

### 4. APIキャッシュ機構

**問題**：レート制限と通信コスト

**解決**：30分ごとのキャッシュ更新

```cpp
if(millis() - lastWeatherUpdate >= 1800000) {
    fetchWeatherFromAPI();
    lastWeatherUpdate = millis();
}
// その間はキャッシュデータを表示
displayFromCache();
```

**効果**：API呼び出し **50%削減**（96→48回/日）

## 🔧 トラブルシューティング

### ❌ Issue: BME280初期化失敗

**原因**：M5Stack固有の設定を無視

**解決**：

```cpp
// ❌ 失敗
Wire.begin(22, 21);
Wire.setClock(400000);

// ✅ 成功
Wire.begin(BME_SDA, BME_SCL);
Wire.setClock(100000);  // 低速クロック
```

---

### ❌ Issue: 関数名衝突エラー

**原因**：`timerStart` はESP32内部関数

**解決**：

```cpp
unsigned long timerStartTime = 0;  // 明確な命名
```

---

### ❌ Issue: アップロード失敗

**症状**：`Failed to connect to ESP32: No serial data received`

**解決**：

1. アップロード開始直後に **BOOTボタン**（左）押下
2. "Connecting..." 中に継続押下
3. "Writing..." 後に離す

**代替案**（platformio.ini）：

```ini
upload_speed = 115200
```

---

### ❌ Issue: メモリ不足

**対策**：

```cpp
// ❌ RAM浪費
String labels[] = {"Temperature", "Humidity"};

// ✅ FLASH配置
const char* labels[] = {"Temp", "Hum"};

// 文字列リテラルをFLASHに配置
showMessage(F("OK"));
```
