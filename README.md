【ざっくり言うと】

M5Stackという小型コンピュータに、各種センサーを接続して、部屋の環境を計測・
表示するプログラムです。

┌─────────────┐
│  M5Stack    │ ← 小型コンピュータ（スマートフォン並みの処理能力）
│  (ESP32)    │
└──────┬──────┘
       │
    3本のセンサーを接続
       │
       ├─ 温湿度センサー（BME280）
       │   └─ 室温・湿度・気圧を計測
       │
       ├─ 照度センサー（NJL7520L）
       │   └─ 部屋の明るさを計測
       │
       └─ 内蔵IMU（加速度・地磁気）
           └─ 傾き・方向を検出
       
       ↓ (計測データを集約)
       
   ┌─────────────┐
   │ 2インチの    │ ← データを画面に表示
   │ディスプレイ │
   └─────────────┘
       ↓ (さらにネットワーク接続)
   
   ┌─────────────────────────┐
   │ OpenWeatherMap API      │ ← インターネット経由で天気情報を取得
   │ (天気予報データベース)  │
   └─────────────────────────┘

【何のためのプログラムか？】

使用シーン: デスク上に置いて、常に環境をチェック

実例：
朝、デスクに置くと...

「あ、今日は湿度75%もある。除湿しよう」
「気温28°C、外は晴れか。冬コートは要らないな」

こういった「ちょっとした判断」に役立つ情報を提供するデバイス

void setup(){
    // M5Stackの初期化
    M5.begin();
    
    // センサー接続確認
    Wire.begin(22, 21);  // I2Cバス初期化
    initBME280();         // 温湿度センサー確認 → 「OK」or「Error」表示
    
    // WiFi接続
    connectWiFi();        // jikei-class-airに接続 → 「WiFi OK」表示
    
    // 時刻同期
    configTime(...);      // NTPサーバーから正確な時刻取得
    
    // 天気データ初期取得
    requestWeatherFetchNow(0);  // 大阪の天気情報をAPI取得
    
    // メニュー画面表示
    drawHomeScreen();
}

【2️⃣ 通常動作（loop関数 - 毎秒繰り返し）】

void loop(){
    // ⓐ ボタン入力チェック
    if(M5.BtnA.wasPressed()){
        // Aボタン: メニュー上へ移動
        menuCursor--;
    }
    if(M5.BtnB.wasPressed()){
        // Bボタン: 選択確定または画面遷移
        screenMode = menuCursor;
    }
    if(M5.BtnC.wasPressed()){
        // Cボタン: メニュー下へ移動
        menuCursor++;
    }
    
    // ⓑ 1秒ごとのセンサー読み取り（UPDATE_INTERVAL=1000ms）
    if(now - lastUpdate >= 1000){
        
        // センサーから値を読み込む
        float temp = bme.readTemperature();   // 温度(°C)
        float hum = bme.readHumidity();       // 湿度(%)
        int lux = analogRead(LIGHT_PIN);      // 照度(0-2000)
        
        // リングバッファに保存（最新60ポイント）
        tempData[dataIndex] = temp;
        humData[dataIndex] = hum;
        luxData[dataIndex] = lux;
        dataIndex = (dataIndex + 1) % 60;
        
        // ⓒ LED自動制御
        if(lux < 100){
            digitalWrite(GPIO25, HIGH);  // 部屋が暗い → LED ON
        } else {
            digitalWrite(GPIO25, LOW);   // 部屋が明るい → LED OFF
        }
        
        // ⓓ 30分ごとの天気API更新
        if(now - lastWeatherUpdate >= 30分){
            requestWeatherFetchNow(cityIndex);  // API呼び出し
            lastWeatherUpdate = now;
        }
        
        // ⓔ 現在の画面を再描画
        if(screenMode == 0){
            drawNormalScreen();      // センサー表示画面
        } else if(screenMode == 1){
            drawGraphScreen();       // グラフ画面
        } else if(screenMode == 3){
            drawWeatherScreenOptimized(cityIndex, false);  // 天気画面
        }
        // ... その他の画面
        
        // ⓕ アイドル処理（5秒以上操作なし）
        if(now - lastInteraction > 5000){
            drawIdleFaceAnimated(temp, hum, lux, ...);  // 顔アニメ表示
        }
    }
}

🎯 初心者が知るべき3つのポイント
================================================================================

【① 「センサーの値は不完全」】

BME280.readTemperature() の返り値
↓
± 1°C の誤差がある可能性

だから「完璧な値」と思わず、
「だいたいこのくらい」と考える必要がある

【② 「APIは遅い」】

インターネット経由の通信 = 500ms～2秒かかる

だから「毎秒呼ぶ」はNG
→ 30分に1回くらいがちょうどいい
→ その間はキャッシュデータを表示

【③ 「画面描画は重い」】

320x240ピクセルのディスプレイ全部塗り直し
↓
毎秒やると負荷が大きい
↓
解決: 差分描画（変わった部分だけ書き直す）

【1. BME280マルチアドレス自動検出】

課題: ハードウェアの個体差でセンサーアドレスが異なる（0x76/0x77/0x75）

解決策:

bool initBME280(){
    byte possibleAddresses[] = {0x76, 0x77, 0x75};
    for(byte i=0; i<3; i++){
        if(bme.begin(possibleAddresses[i], &Wire)){
            showTempMessage("BME OK", 700);
            return true;
        }
    }
    showTempMessage("BME Error", 900);
    return false;
}

効果: ハードウェアバリエーション対応、デバッグ時間短縮

【2. 差分描画による最適化】

課題: 小型ディスプレイへの連続描画は負荷が高い

解決策 - 3段階の更新戦略:

// ステップ1: 都市切り替え時は全画面再描画
if(force || lastDisplayedCityIndex != cityIdx){
    drawWeatherFullFromCache(cityIdx);
    return;
}

// ステップ2: 説明文変更のみ部分更新
if(c.description != lastDisplayedDescription){
    M5.Lcd.fillRect(20, 80, 280, 40, NORMAL_BG);
    M5.Lcd.setCursor(20, 85);
    M5.Lcd.printf("%s", c.description.c_str());
}

// ステップ3: 温度0.1°C以上の変化のみ更新
if(fabs(c.temp - lastDisplayedTemp) >= 0.1f){
    M5.Lcd.fillRect(70, 140, 200, 80, NORMAL_BG);
    M5.Lcd.setCursor(70, 140);
    M5.Lcd.printf("%.1f C", c.temp);
}

効果: 描画負荷 60-80%削減、ちらつき完全排除

【3. リングバッファによる効率的データ管理】

課題: 60ポイント × 3系列（温度・湿度・照度）のデータ蓄積

解決策:

constexpr int MAX_DATA_POINTS = 60;
float tempData[MAX_DATA_POINTS];
int dataIndex = 0;

// 新データ追加 - O(1)で完了
tempData[dataIndex] = temp;
dataIndex = (dataIndex + 1) % MAX_DATA_POINTS;

// 最新データ取得
int prevIndex = (dataIndex + MAX_DATA_POINTS - 1) % MAX_DATA_POINTS;
float latestTemp = tempData[prevIndex];

効果:
- 固定メモリ使用（RAM効率化）
- O(1)アクセス時間
- ヒープフラグメンテーション回避

【4. OpenWeatherMap APIキャッシュ機構】

課題: API呼び出しレート制限、通信コスト削減

実装:

struct WeatherCache {

// 30分ごとの更新制御

効果:
- API呼び出し96回/日 → 48回/日（50%削減）
- 6都市キャッシュで都市切り替え時も高速表示

【5. アイドル時のインタラクティブ顔アニメーション】

要素:
- 体の揺れ: sin/cos波による自然な揺動
- 目の移動: 3秒ごとランダム移動（±12px, ±6px）
- まばたき: 4秒間隔、180ms継続
- 口の感情変化: 温度・湿度・天気に応じた3パターン
- びっくり表情: 加速度1.5G以上で1.5秒間表示

技術仕様:

float bodySwayX = sin(millis()/2000.0) * 12.0;
float bodySwayY = cos(millis()/2500.0) * 6.0;

if(temp > 30 || weatherSymbol == "SUN"){
    M5.Lcd.fillEllipse(faceX, faceY+40, mouthW/2, mouthH, RED);  // 開口
} else if(hum > 70 || weatherSymbol == "RAIN"){
    M5.Lcd.drawLine(faceX-mouthW/2, faceY+40, faceX+mouthW/2, faceY+40, RED);  // 直線
}

特殊設計: コンパス画面のみアイドル待機30秒（他は5秒）
- 理由: 方位確認は継続して見る用途が多い

【📌 Issue #1: BME280初期化失敗】

症状:
sensor init failed
bme.begin() return false

原因: M5Stack固有のI2C設定を無視した一般的なArduino手法

解決:

// ❌ 失敗パターン
Wire.begin(22, 21);
Wire.setClock(400000);  // M5Stackは高速クロック非対応

// ✅ 成功パターン
Wire.begin(BME_SDA, BME_SCL);
Wire.setClock(100000);  // 低速クロック指定

教訓: プラットフォーム固有の制約を理解してからコーディング開始すべき

【📌 Issue #2: ESP32内部関数名衝突】

症状:
error: reference to 'timerStart' is ambiguous

原因:
unsigned long timerStart = 0;  // ESP32内部関数と同名

解決:
unsigned long timerStartTime = 0;  // 明確な命名に変更

【📌 Issue #3: アップロード失敗（シリアル通信タイムアウト）】

症状:
Failed to connect to ESP32: No serial data received

解決手順:
1. アップロード開始直後にBOOTボタン（左ボタン）押下
2. "Connecting..." 中に継続
3. "Writing..." 表示後に離す

代替案 (platformio.iniに追加):
upload_speed = 115200

【📌 Issue #4: メモリ不足警告】

状態:
Flash: 86.9% (1139149 bytes from 1310720 bytes)

最適化手法:

// 悪い例: RAM浪費
String labels[] = {"Temperature", "Humidity", "Pressure"};

// 良い例: FLASH配置
const char* labels[] = {"T", "H", "P"};

// 文字列リテラル最小化
showTempMessage(F("OK"), 700);  // F()マクロ使用

    └─ IMU: 加速度・地磁気

[リングバッファに蓄積]
    ├─ tempData[60]
    ├─ humData[60]
    ├─ luxData[60]
    └─ dataIndex = (dataIndex+1) % 60
    ↓
[画面レンダリング]
    ├─ Sensor View: リアルタイム値

    ├─ Statistics: 平均・最大・最小

    └─ OpenWeatherMap ── キャッシュ ── Weather画面
